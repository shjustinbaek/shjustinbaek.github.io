## Problem Definition

https://leetcode.com/problems/reverse-integer/

 > Given a signed 32-bit integer `x`, return `x` *with its digits reversed*. If reversing `x` causes the value to go outside the signed 32-bit integer range `[-231, 231 - 1]`, then return `0`.
 >
 > **Assume the environment does not allow you to store 64-bit integers (signed or unsigned).**
 >
 >  
 >
 > **Example 1:**
 >
 > ```
 > Input: x = 123
 > Output: 321
 > ```
 >
 > **Example 2:**
 >
 > ```
 > Input: x = -123
 > Output: -321
 > ```
 >
 > **Example 3:**
 >
 > ```
 > Input: x = 120
 > Output: 21
 > ```
 >
 > **Example 4:**
 >
 > ```
 > Input: x = 0
 > Output: 0
 > ```
 >
 >  
 >
 > **Constraints:**
 >
 > - `-231 <= x <= 231 - 1`

## Initial Solution : $O(1)$

```python
class Solution:
    def reverse(self, x: int) -> int:
        rtn = None
        if x >= 0:
            rtn = int(str(x)[::-1])
        else:
            x *= -1 
            rtn = -1*int(str(x)[::-1])
        limit = 2**31
        if rtn >= limit or rtn < limit*-1:
            return 0
        else:
            return rtn
```

The problem states that the environment can only store 32bit signed integer.  Therefore we should check 32bit overflow after we reverse the integer, since the reversed number is sent to the environment.
